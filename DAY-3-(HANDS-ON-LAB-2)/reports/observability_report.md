# 🔍 Observability & Audit Trail Report

## 📖 Introduction
Observability is a critical pillar for any production AI system. This report outlines how we maintain visibility, tracking, and debugging capabilities within the **Customer Support Query Router** application without cluttering the frontend user experience.

## 🛠 Tracking Mechanisms

### 1. Backend Audit Log (`backend_audit.log`)
We implemented standard Python logging to capture an immutable audit trail. All critical node executions, routing decisions, and exceptions are written into a physical log file (`backend_audit.log`) inside the root directory.

#### What gets logged?
- **Workflow Triggers:** Timestamps of when a workflow execution starts.
- **Node Execution Analytics:** Insights into which LangGraph nodes are executing and their respective input query and inference results.
- **Model Confidence Scores:** Captures the raw float values generated by the Hugging Face Pipeline. 
- **Warnings & Guardrails:** Anytime the user inputs empty or invalid fields, a warning is captured that the guardrail successfully deflected the query.
- **Failures:** Tracks exceptions where model instantiation fails or a fallback path is traversed.

### 2. LangGraph State Inspection
State object transparency is provided at the UI level for auditing purposes. By expanding the **Show Graph State (Audit Trace)** widget in the UI, developers or administrators can see the exact JSON object payload returned by the graph, including:
- Original query
- Classified sentiment
- Extracted confidence
- Final routing decision

## 🛡 System Guardrails
To prevent token waste or model inference exceptions, pre-LLM and pre-Model guardrails are injected:
1. **Frontend UI Check**: Disables the pipeline execution if `user_query` is empty.
2. **Backend Node Check**: Node `analyze_sentiment_node` asserts `len(query.strip()) >= 3`.

These proactive validation techniques ensure robustness while saving compute overhead on garbage data.
